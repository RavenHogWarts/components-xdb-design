# 星露谷打卡多视图方案设计

本方案旨在为 `stardewHabit` 扩展“种植”、“图鉴”、“统计”三个主要视图，提升打卡的可视化乐趣与历史数据追踪能力。

---

## 1. 界面与交互架构

### 1.1 Toolbar (导航栏) 改造
- **选项卡组合 (TabGroup)**：在 Brand 标志和日期选择器之间，新增一组木质按钮页签：
  - **种植 (Planting)**：核心农场画面，继承现有的卡片网格和天空渐变。
  - **图鉴 (Almanac)**：星露谷百科全书，展示所有作物的生命成长链，支持按名称/ID检索。
  - **统计 (Stats)**：打卡表现大盘与农场种植成果归档。
- **日期选择器显隐策略**：
  - “种植”视图与当前选择日期强关联（可补卡或查看旧的生长进度），故仅在“种植”视图展示右侧的 `input[type="date"]`。
  - “图鉴”和“统计”视图展示的是全局只读属性或聚合历史数据，切入后隐藏顶栏的日期选择器，保持界面清爽。

### 1.2 视图层设计 (View Switching)
通过在宿主 React 组件中维护一个 `currentView` 状态：
```typescript
type ViewType = 'planting' | 'almanac' | 'stats';
const [currentView, setCurrentView] = useState<ViewType>('planting');
```
在主渲染区域进行条件分流渲染，并复用从 `props` 传入的全局 `app`, `moment`, `viewData`, `viewDefinition` 和通过 reconciliation 算出的 `habitsData` 状态。

## 2. 详细功能模块设计

### 2.1 “图鉴”视图 (Almanac) ── 作物生长链

图鉴展示星露谷 50 余种作物的生命轨迹。

- **搜索与过滤**：在图鉴顶部增加文本搜索框。用户可输入作物中文名称或种子 ID，进行过滤渲染，提高查找效率。
- **成长阶段链展示**：
  - 每个作物渲染为一个横向卡片（使用淡黄色羊皮纸 `#fcf2d9` 材质）。
  - 从左到右渲染该作物的全部生命阶段，包含：种子状态 $\rightarrow$ 生长阶段 1..N $\rightarrow$ 成熟态 $\rightarrow$ 收获成果态。
  - 每一个阶段下方用小字显示累计打卡/浇水天数：
    - 阶段 0（种子）：`0天`
    - 生长阶段 $i$：利用 `daysInPhase` 数组进行前缀和累加，计算当前阶段处于哪几天区间。例如初芽阶段显示为 `第1天`，中生代显示为 `第2-3天`。
    - 成熟期阶段：显示为 `${growthDays}+天`（例如 `4+天`）。
    - 丰收/一次性收获态：作为生长链的终点，在成熟后呈现。

### 2.2 “统计”视图 (Stats) ── 打卡与收获成果汇总

统计视图由打卡表现与农场成果两大卡片块组成：

#### A. 打卡总体表现详情
1. **累计打卡总天数**：所有习惯在单文件记录里，打卡成功的任务数累计。
2. **总完成率**：`总完成任务数 / 总打卡记录任务数 * 100%`。
3. **单项习惯卡片**：
   - 习惯名称与作物图标。
   - 累计打卡天数。
   - **当前连续打卡 (Current Streak)**：根据选中日期的完成情况向前追溯的连续天数。
   - **历史最长连续打卡 (Max Streak)**：
     - **算法**：过滤出当前习惯所有 `isDone: true` 的任务日期，去重后从小到大升序排序。遍历这些日期，比较相邻日期之间的差值。若差值为 1，则累加当前连击值；若大于 1，则证明连续打卡中断，记录当前值并将其重置为 1。最终取遍历过程中的最大值。

#### B. 农场作物成果汇总
1. **当前种植中作物**：
   - 从 `habitsData` 的 `metadata.current_crop` 提取正在成长的作物。
   - 显示作物图像、当前已浇水天数和总天数（例如 `2/4天`），以及种植开始日期。
2. **已收获作物成果**：
   - 汇总各单文件中 `crop_history` 里 `status === 'harvested'` 的条目。
   - 统计每种作物的收获次数，并使用 `cropsSprite` 将对应作物的丰收小图标渲染出来，旁边标注 “已收获 X 次”。
3. **已枯萎作物成果**：
   - 汇总 `crop_history` 里 `status === 'withered'` 的条目。
   - 以暗淡灰阶（或枯焦）像素图标渲染，并标注 “已枯死 Y 次”。
4. **种植成功率**：
   - 计算公式：`总收获次数 / (总收获次数 + 总枯萎次数) * 100%`。

## 3. 边界条件与极端情况处理

| 边界场景 | 表现与防护策略 |
| :--- | :--- |
| **空数据库/新用户** | 此时 `crop_history` 和 `tasks` 为空。在统计视图中，“种植成功率”分子分母均为 0，应捕获并展示为 `0%`（防止 NaN 崩溃）；打卡大盘展示“暂无打卡记录”，成果卡片显示“农场尚无种植和收获成果，快去种植吧！”等提示。 |
| **作物未在 `Crops.json` 注册** | 如果用户强行修改了打卡文件中的作物 ID 导致未知作物，在图鉴或统计中将使用 `Crops.json` 中的默认作物（防风草 id='24'）进行降级兜底，避免发生渲染 Null 指针报错。 |
| **大量作物收获记录性能瓶颈** | 如果打卡时间超过数年，`crop_history` 会很长。在统计汇总时，聚合 Map 累加属于 $O(N)$ 复杂度，不会出现性能瓶颈。图鉴视图有 50 余个作物，进行按名过滤属于轻量级过滤，渲染流畅。 |
| **时区与日期转换** | 均通过 Obsidian 注入的 `props.moment` 进行计算，确保与宿主软件时区及自定义日期格式（`dateFormat`）完美吻合。 |

## 4. 样式及视觉标准 (Style Guide)

- **主配色**：采用星露谷代表性的暖褐色/原木色（主面板 `#f7e0b5`，边框 `#5a3c20`），内侧阴影采用 `#d8a065`。
- **页面选项卡**：`.stardewHabit--TabGroup`。未激活时，颜色比 Toolbar 底色稍深，略显下沉；激活时使用高亮橙色，带向上偏移，突出当前页面。
- **羊皮纸图鉴背景**：图鉴使用星露谷笔记本的复古羊皮纸质感背景，字重加大，带轻微木纹边框。
- **动态呼吸**：对按钮悬停和 Tab 切换加入 $0.1s$ 的微缩动画，增强用户交互动感。
